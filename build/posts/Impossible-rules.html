Impossible rules

I thought I'd collect some rules I'd like to see that are impossible with the current ESLint. For posterity.

  • Prefer RegExp#exec() over String#match() - Requires type analysis.
  • Prefer .includes() over .indexOf() - While it's possible today with some assumption. It's not perfect and would be improved with some type analysis.
  • Prefer .startsWith()/.endsWith() over regex/indexOf/slice/etc - Only the regex case is possible today. With type analysis, we could do all the cases in that issue.
  • Require braces for arrow functions that doesn't return anything. Would need to know the return value of any called function in the arrow function body. ##
